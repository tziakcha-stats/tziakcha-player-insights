# 收藏功能设计

## 背景

当前脚本已在 `/game`、`/record`、`/history`、`/user/tech` 等页面按路由注入功能，但缺少一个统一的本地收藏能力。用户希望在站点主页顶栏新增一个 `收藏` 一级入口，用于集中保存对局牌谱（`game`）和小局牌谱（`record`），并支持标签管理、筛选以及导入导出。

首版目标是补齐本地收藏闭环，不引入后端依赖，也不要求账号同步。

## 需求范围

### In Scope

- 在站点主页顶栏现有 tab 后新增 `收藏` 一级入口。
- 新增一个由脚本接管渲染的收藏页。
- 收藏页内部包含两个子 tab：
  - `对局收藏`
  - `小局收藏`
- 仅支持从详情页入口加入收藏：
  - `/game/...` 页面收藏整场对局
  - `/record/...` 页面收藏当前小局
- 收藏项支持维护多个标签。
- 收藏页支持按标签筛选。
- 收藏数据保存在本地。
- 支持导出为 JSON 文件。
- 支持从 JSON 文件导入，并在重复项上执行标签合并去重。

### Out of Scope

- 从历史列表、对局列表直接收藏。
- 云端同步、跨设备同步。
- 收藏夹概念。
- 复杂搜索语法、排序自定义、批量编辑标签。
- 标签外的长文本备注编辑器。

## 用户流程

### 收藏对局

1. 用户进入 `/game/...` 页面。
2. 页面出现 `加入收藏` 或 `已收藏` 按钮。
3. 用户点击按钮后，可在轻量编辑区添加或修改标签。
4. 保存后写入本地存储，并即时反馈结果。

### 收藏小局

1. 用户进入 `/record/...` review 页面。
2. 页面出现 `加入收藏` 或 `已收藏` 按钮。
3. 用户编辑标签并保存。
4. 保存后写入本地存储，并即时反馈结果。

### 浏览收藏

1. 用户点击主页顶栏新增的 `收藏` tab。
2. 进入收藏页后，默认进入上次停留的子 tab；若无记录则进入 `对局收藏`。
3. 用户可切换 `对局收藏` / `小局收藏` 子 tab。
4. 用户可选择一个或多个标签进行筛选。
5. 用户可通过搜索框按标题、摘要、标签进行前端模糊匹配。
6. 用户可点击条目返回原始牌谱页面。

### 导出与导入

1. 用户在收藏页点击 `导出`，下载当前本地收藏 JSON 文件。
2. 用户点击 `导入` 并选择 JSON 文件。
3. 系统校验格式。
4. 对重复收藏项执行合并：
   - 以 `type + id` 作为去重键。
   - 标签取并集并去重。
   - 展示字段优先保留信息更完整的一侧。
5. 导入完成后展示结果摘要：新增条数、合并条数、失败条数。

## 路由与界面设计

## 一级导航

- 在站点主页顶栏的原有导航项后追加一个 `收藏` 入口。
- 点击后跳转到站点主页上的约定 hash 路由 `#reviewer-favorites`。
- 选择 hash 路由而不是全新 pathname，避免站点后端无法识别新路径导致刷新 404。
- 该路由由脚本检测 hash 变化后接管页面主体渲染。
- 顶栏入口应将用户带回站点主页并附加该 hash；若当前已在主页，则仅更新 hash。

## 收藏页结构

收藏页由以下区域组成：

1. 顶部工具栏
   - `导入`
   - `导出`
   - 搜索框
2. 标签筛选栏
   - 展示当前子 tab 下的全部标签
   - 支持多选筛选
3. 子 tab 区
   - `对局收藏`
   - `小局收藏`
4. 收藏列表
   - 标题
   - 摘要信息
   - 标签 chips
   - 原始链接
   - 编辑标签入口
   - 取消收藏入口
5. 空状态
   - 无收藏
   - 当前筛选条件下无结果

## 详情页入口设计

### 对局页

- 在 `/game/...` 页面现有脚本注入区域附近新增收藏按钮。
- 未收藏时显示 `加入收藏`。
- 已收藏时显示 `已收藏`。
- 点击后展开标签编辑 UI，支持输入、删除、保存。

### 小局页

- 在 `/record/...` review 控制区新增同样的收藏入口。
- 行为与对局页保持一致，但数据类型为 `record`。

## 数据模型

本地存储采用单 key 版本化 JSON 文档。

### Storage Key

- `tzi-reviewer:favorites`

### Document Shape

```json
{
  "version": 1,
  "updatedAt": "2026-03-25T00:00:00.000Z",
  "ui": {
    "activeTab": "game"
  },
  "games": [],
  "records": []
}
```

### Favorite Item Shape

```json
{
  "id": "stable-id",
  "type": "game",
  "sourceUrl": "https://example.com/game/123",
  "title": "四人南 / 2026-03-25",
  "summary": "玩家A、玩家B、玩家C、玩家D",
  "tags": ["练习", "高打点"],
  "createdAt": "2026-03-25T00:00:00.000Z",
  "updatedAt": "2026-03-25T00:00:00.000Z"
}
```

### 字段约束

- `id`: 来自页面可稳定解析的业务标识。
- `type`: `game` 或 `record`。
- `sourceUrl`: 原始页面 URL，用于回跳。
- `title`: 用于列表展示的人类可读标题。
- `summary`: 简短摘要；允许为空字符串。
- `tags`: 标签字符串数组。
- `createdAt`: 首次收藏时间。
- `updatedAt`: 最近一次标签更新或导入合并时间。

### 标签标准化

- 去首尾空格。
- 过滤空字符串。
- 大小写保持原样，不做自动转小写。
- 去重。
- 限制单标签最大长度为 20 个字符，避免异常输入破坏 UI。
- 单个收藏项最多保留 20 个标签；超出部分在保存或导入时丢弃。

## 数据来源与解析

### 对局收藏

- 从 `/game/...` 页面提取稳定 `game id`。
- 生成展示标题与摘要：
  - 标题优先使用页面上可读对局标题或场次信息。
  - 摘要可包含日期、玩家名、局数等现有页面可直接读取的信息。

### 小局收藏

- 从 `/record/...` 页面提取稳定 `record id`。
- 标题优先使用当前局次或牌谱标题。
- 摘要可包含场次、玩家名、手数等已有可见信息。

### 失败降级

- 若无法解析稳定 id，则不显示收藏按钮。
- 若能解析 id 但无法提取完整标题/摘要，则允许降级为最小可用展示文本。

## 存储与导入导出逻辑

## 本地仓库职责

需要一个独立的 favorites 仓库层，负责：

- 读取与写入本地存储。
- 处理版本迁移。
- 根据 `type + id` 查询收藏状态。
- 新增或更新收藏项。
- 删除收藏项。
- 规范化标签。
- 导入合并。
- 导出序列化。

## 导入合并规则

- 去重键：`type + id`
- 若不存在则新增。
- 若已存在则执行合并：
  - 标签做并集去重。
  - `title`、`summary`、`sourceUrl` 的字段合并规则固定为：
    - 若一侧为空，使用非空一侧。
    - 若两侧都非空，保留字符长度更长的一侧。
    - 若字符长度相同，优先保留本地已有值。
  - `createdAt` 保留较早时间。
  - `updatedAt` 更新为当前合并时间。

## 搜索与筛选语义

- 搜索使用大小写不敏感的子串匹配。
- 搜索范围为 `title`、`summary` 与所有 `tags` 拼接后的文本。
- 不做空格分词，不做拼音或模糊纠错。
- 输入为空字符串时，视为不过滤。
- 标签筛选与搜索同时存在时，结果取交集。
- 多标签筛选采用“全部命中”语义，即收藏项必须同时包含所有已选标签。

## 错误处理

- `localStorage` 不可用时：
  - 收藏按钮显示为禁用态，不隐藏。
  - 收藏页显示不可用提示，不渲染导入导出与编辑操作。
- 导入 JSON 解析失败时：
  - 不修改现有数据。
  - 给出错误提示。
- 导入格式不匹配时：
  - 跳过非法条目。
  - 在结果摘要中统计失败数。

  - record 页 id/标题/摘要提取

## 实现拆分建议

建议按以下模块拆分，避免把逻辑压进单一入口文件：

- 路由识别：新增 favorites hash 页面判断。
- 导航注入：负责顶栏追加 `收藏` tab。
- 仓库层：负责本地数据读写与导入导出。
- 解析层：负责从 game/record 页面提取收藏元数据。
- 页面入口按钮：对局页、小局页分别接入收藏 UI。
- 收藏页渲染：负责一级页面 UI、子 tab、筛选、列表、导入导出。

## 风险与取舍

### 风险

- 站点顶栏 DOM 结构可能不稳定，导航注入需要重试与幂等保护。
- `/game` 与 `/record` 页面中稳定 id 的提取方式可能并不统一，需要先验证 DOM 与 URL。
- `localStorage` 容量有限，但首版收藏量通常足够；若后续引入大量字段再考虑升级。

### 当前取舍

- 首版只做本地存储，不做云同步。
- 首版只做标签，不做独立收藏夹。
- 首版只支持详情页入口，不做列表页快速收藏。
- 首版导出格式以内部稳定 JSON 为主，不额外兼容第三方格式。
